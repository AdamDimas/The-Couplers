Middle Man

Middle Man adalah terjadi ketika sebuah kelas berperan sebagai perantara yang hanya meneruskan panggilan atau delegasi ke kelas lain tanpa menambahkan nilai tambah atau logika yang signifikan. Dalam situasi ini, middle man tidak melakukan banyak pekerjaan sendiri, tetapi hanya meneruskan panggilan ke kelas lain, yang bisa menyebabkan overhead yang tidak perlu dan meningkatkan kompleksitas kode.

Kenapa bisa terjadi "Middle Man"?


Smell ini bisa jadi hasil dari penghapusan Rantai Pesan yang terlalu bersemangat. Dalam kasus lain, ini bisa jadi hasil dari pekerjaan yang berguna dari sebuah kelas yang secara bertahap dipindahkan ke kelas lain. Kelas tetap sebagai sebuah shell kosong yang tidak melakukan apa pun selain mendelegasikan.

Cara Mengatasi

Jika sebagian besar kelas metode didelegasikan ke kelas lain, Hapus Middle Man adalah yang paling tepat.

Contoh Code

Sebelum Refactor
                    
public class MiddleMan {
    private Service service;

    public MiddleMan() {
        this.service = new Service();
    }

    public void doSomething() {
        service.performAction();
    }

    public void doAnotherThing() {
        service.performAnotherAction();
    }
}

public class Service {
    public void performAction() {
        // Logic for performing action
    }

    public void performAnotherAction() {
        // Logic for performing another action
    }
}

Dalam contoh di atas, kelas MiddleMan bertindak sebagai perantara yang hanya meneruskan panggilan ke kelas Service tanpa menambahkan nilai tambah atau logika tambahan. Hal ini menyebabkan kelas MiddleMan menjadi tidak diperlukan dan memperkenalkan overhead yang tidak perlu.

Setelah Refactor
                    
public class ServiceClient {
    private Service service;

    public ServiceClient() {
        this.service = new Service();
    }

    public void doSomething() {
        service.performAction();
    }

    public void doAnotherThing() {
        service.performAnotherAction();
    }
}

public class Service {
    public void performAction() {
        // Logic for performing action
    }

    public void performAnotherAction() {
        // Logic for performing another action
    }
}

Setelah refaktorisasi, kita menghapus kelas MiddleMan dan memindahkan logika penggunaan kelas Service langsung ke kelas ServiceClient. Sekarang, kelas ServiceClient langsung berinteraksi dengan kelas Service, menghilangkan overhead dan kompleksitas yang disebabkan oleh middle man. Kode menjadi lebih bersih dan lebih langsung dalam penggunaan layanan.
